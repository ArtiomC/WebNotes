UNDOING CHANGES

To undo changes we need to know on which level (на каком уровне) we have the changes we want to undo are located.

For example if we have a new file which was not added to the staging area (index) and we did not commit or push it. There is a file and we need to get rid of it.

- Working directory (рабочая дериктория в которой уже есть измененные файлы)

While the changes or a file are in working directory, the below commands are allowed to be used.

  • git checkout --file.txt
  • git checkout .
  • git clean -xdf

There are files in the working directory and the working tree is clean.

Changes were made in the files which have already existed in the local folder. The same files are in the local folder, however the git status of the files has changed. The files have been modified

 

By using the git restore . or git checkout . commands the changes made to the files can be undone. The . tells the terminal to use the command on all files in the folder. Please refer to the below images. 

Important

If you do a checkout . (or restore .) all of the changes in the files affected with the command will be lost permanently. Be careful with that.
